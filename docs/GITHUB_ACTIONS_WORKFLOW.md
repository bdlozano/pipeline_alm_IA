# GitHub Actions Workflow: CI Failure Detection\n\n## 📋 Descripción General\n\nEste workflow de GitHub Actions:\n\n1. **Detecta fallos** en pipelines de CI/CD\n2. **Captura contexto** (logs, branch, commit, autor)\n3. **Crea issues automáticamente** con información completa\n4. **Evita duplicados** verificando issues existentes\n5. **Filtra ramas** para evitar test branches\n6. **Notifica al agente IA** via webhook (opcional)\n\n## 🎯 Características Principales\n\n### ✅ Idempotencia (Anti-duplicados)\n\n```yaml\n# Verificación 1: Filtrado de Ramas\nbranch_filter:\n  ALLOWED_BRANCHES: [\"main\", \"master\", \"develop\", \"release/*\"]\n  REJECTED_PATTERNS: [\"test/\", \"temp/\", \"wip/\", \"feature/ai-fix\", \"fix/ai-heal\"]\n  → Evita crear issues en ramas de test\n\n# Verificación 2: Búsqueda de Issues Duplicados\ndeduplicate:\n  - Busca issues abiertos con el mismo workflow y branch\n  - Comprueba si el commit es el mismo\n  - Si existe issue < 5 minutos atrás → Skip\n  → Previene múltiples issues para el mismo error\n```\n\n### 📊 Flujo de Decisión\n\n```\nCI Pipeline FAIL\n    ↓\n¿Conclusion = failure?\n    ├─ NO  → STOP\n    └─ SÍ ↓\n\n¿Branch en ALLOWED_BRANCHES?\n    ├─ NO  → STOP (rama de test)\n    └─ SÍ ↓\n\n¿Branch no en REJECTED_PATTERNS?\n    ├─ NO  → STOP (rama temporal)\n    └─ SÍ ↓\n\n¿Existe issue duplicado (< 5 min)?\n    ├─ SÍ  → STOP (idempotencia)\n    └─ NO ↓\n\n✅ CREATE ISSUE\n    ↓\n🔔 NOTIFY AGENT (webhook)\n```\n\n## 📝 Desglose de Pasos\n\n### Paso 1: Extract failure context\n\n**Responsabilidad**: Capturar metadata del workflow que falló\n\n```yaml\nsteps:\n  - name: Extract failure context\n    id: context\n    run: |\n      echo \"workflow_name=${{ github.event.workflow_run.name }}\" >> $GITHUB_OUTPUT\n      echo \"branch=${{ github.event.workflow_run.head_branch }}\" >> $GITHUB_OUTPUT\n      echo \"commit_sha=${{ github.event.workflow_run.head_commit.id }}\" >> $GITHUB_OUTPUT\n```\n\n**Salida**:\n- `workflow_name`: Nombre del workflow que falló (ej: \"Tests\")\n- `branch`: Rama donde falló (ej: \"main\")\n- `commit_sha`: Hash del commit (ej: \"abc123def456\")\n- `actor`: Usuario que triggereó el workflow\n\n### Paso 2: Filter production branches\n\n**Responsabilidad**: Asegurar que solo procesamos fallos en ramas importantes\n\n```yaml\n# ✅ PERMITIDAS (auto-healing activo)\nALLOWED_BRANCHES:\n  - main         # Rama principal\n  - master       # Rama principal (alternativa)\n  - develop      # Rama de desarrollo\n  - release/*    # Ramas de release\n\n# ❌ RECHAZADAS (auto-healing desactivo)\nREJECTED_PATTERNS:\n  - test/        # Ramas de prueba\n  - temp/        # Ramas temporales\n  - wip/         # Work-in-progress\n  - feature/ai-fix  # Ya generada por el agente\n  - fix/ai-heal     # Ya generada por el agente\n```\n\n**Lógica**:\n```bash\nif branch matches REJECTED_PATTERN:\n  SHOULD_PROCESS = false\nelse if branch not in ALLOWED_BRANCHES:\n  SHOULD_PROCESS = false\nelse:\n  SHOULD_PROCESS = true\n```\n\n### Paso 3: Download logs & artifacts\n\n**Responsabilidad**: Obtener logs de los jobs que fallaron\n\n```yaml\n- name: Download logs from workflow run\n  id: logs\n  uses: actions/github-script@v7\n```\n\n**Proceso**:\n1. Listar todos los jobs del workflow que falló\n2. Para cada job con `conclusion = 'failure'`:\n   - Descargar sus logs\n   - Limitar tamaño a 3000 caracteres\n   - Guardar en archivo `ci-logs.txt`\n3. Si no hay logs disponibles → usar texto por defecto\n\n**Output**:\n```\n## Job: test-unit\n\n```\nFAILED tests/test_calculator.py::test_add - AssertionError: expected 5 to equal 10\n```\n```\n\n### Paso 4: Check for existing issues (Deduplication)\n\n**Responsabilidad**: Implementar idempotencia mediante búsqueda de duplicados\n\n```yaml\n- name: Check for existing issues\n  id: deduplicate\n  uses: actions/github-script@v7\n```\n\n**Búsqueda**:\n```javascript\nquery = {\n  q: 'is:issue is:open label:self-healing-candidate in:title \"Tests\" branch:main',\n  sort: 'created',\n  order: 'desc',\n  per_page: 10\n}\n```\n\n**Lógica de deduplicación**:\n```javascript\nfor (issue of searchResults) {\n  createdAt = parse(issue.created_at);\n  \n  // ¿Fue creado hace menos de 5 minutos?\n  if (createdAt > fiveMinutesAgo) {\n    // ¿Es para el mismo commit?\n    if (issue.body.includes(commitSha)) {\n      // → SKIP: Duplicate encontrado\n      duplicate_found = true;\n      break;\n    }\n  }\n}\n```\n\n**Outputs**:\n- `duplicate_found`: \"true\" | \"false\" | \"unknown\"\n- `duplicate_issue_number`: Número del issue duplicado (si existe)\n\n**Ventajas**:\n- ✅ Previene spam de issues\n- ✅ Respeta la ventana de 5 minutos (permite issues para errores distintos)\n- ✅ Se basa en commit SHA (no solo en mensaje)\n\n### Paso 5: Get failed job details\n\n**Responsabilidad**: Identificar el job específico que falló\n\n```yaml\n- name: Get failed job information\n  id: failed_job\n  uses: actions/github-script@v7\n```\n\n**Salida**:\n- `job_name`: \"test-unit\" | \"build\" | etc\n- `job_id`: ID numérico del job\n- `job_url`: URL directo al job en GitHub UI\n\n### Paso 6: Create GitHub issue\n\n**Responsabilidad**: Crear el issue con toda la información\n\n```yaml\n- name: Create GitHub issue for CI failure\n  id: create_issue\n  uses: actions/github-script@v7\n```\n\n**Título del issue**:\n```\n🔴 Self-Healing: test-unit failed on branch main\n```\n\n**Descripción** (template):\n```markdown\n## CI Failure Report\n\n**Status:** ❌ FAILED\n\n### Failure Details\n- **Workflow:** Tests\n- **Job:** test-unit\n- **Branch:** `main`\n- **Commit:** `abc123d`\n- **Author:** @developer\n- **Run URL:** [View Run](https://github.com/...)\n\n### Build Logs\n\n```\nFAILED tests/test_calculator.py::test_add - AssertionError: expected 5 to equal 10\n```\n\n### Auto-Healing Status\n\nAn AI agent will attempt to:\n1. 🔍 Analyze the error logs\n2. 🧠 Identify the root cause\n3. 💡 Generate a fix\n4. 🔀 Create a pull request\n5. ✅ Validate with tests\n```\n\n**Labels asignadas**:\n- `self-healing-candidate` - Candidato para auto-healing\n- `auto-healing` - Sistema de auto-healing activado\n- `ci-failure` - Tipo: fallo de CI\n- `status: pending-analysis` - Estado actual\n\n**Outputs**:\n- `issue_number`: 1234\n- `issue_url`: https://github.com/bdlozano/pipeline_alm_IA/issues/1234\n\n### Paso 7: Notify AI Agent via webhook\n\n**Responsabilidad**: Notificar al agente IA mediante webhook\n\n```yaml\n- name: Notify AI Agent via webhook\n  if: env.SELF_HEALING_WEBHOOK_URL != ''\n  env:\n    SELF_HEALING_WEBHOOK_URL: ${{ secrets.SELF_HEALING_WEBHOOK_URL }}\n    SELF_HEALING_WEBHOOK_SECRET: ${{ secrets.SELF_HEALING_WEBHOOK_SECRET }}\n```\n\n**Payload del webhook**:\n```json\n{\n  \"event\": \"ci_failure\",\n  \"repository\": \"pipeline_alm_IA\",\n  \"owner\": \"bdlozano\",\n  \"issue_number\": 1234,\n  \"workflow\": \"Tests\",\n  \"branch\": \"main\",\n  \"commit_sha\": \"abc123def456\",\n  \"timestamp\": \"2026-06-16T14:30:00Z\"\n}\n```\n\n**Validación de Signature**:\n```javascript\nsignature = HMAC_SHA256(\n  payload,\n  SELF_HEALING_WEBHOOK_SECRET\n)\n// Enviado en header: X-Signature: sha256=<signature>\n```\n\n**Configuración requerida en GitHub Secrets**:\n```\nSELF_HEALING_WEBHOOK_URL = https://agent.example.com/webhooks/ci-failure\nSELF_HEALING_WEBHOOK_SECRET = your-secret-key\n```\n\n## 🔐 Idempotencia Garantizada\n\n### Mecanismo 1: Branch Filtering\n\n```yaml\nIF branch NOT IN allowed_branches OR branch IN rejected_patterns:\n  → EXIT (no crear issue)\n```\n\n**Efecto**: \n- Evita crear múltiples issues para la misma rama\n- Previene issues en ramas de prueba\n\n### Mecanismo 2: Duplicate Detection\n\n```yaml\nSEARCH issues WHERE:\n  - state = open\n  - label = self-healing-candidate\n  - created_at > NOW - 5 minutes\n  - body contains commit_sha\n\nIF found:\n  → EXIT (no crear issue)\n```\n\n**Efecto**:\n- Previene spam en caso de retrigger del workflow\n- Permite múltiples issues si son errores diferentes\n- Ventana de 5 minutos es configurable\n\n### Mecanismo 3: Workflow Filtering\n\n```yaml\nON workflow_run:\n  - workflows: ['Tests', 'Integration Tests', 'Build']\n  - types: [completed]\n  - IF conclusion = 'failure'\n```\n\n**Efecto**:\n- Solo procesa workflows específicos\n- Solo procesa al completarse (no a mitad)\n- Solo procesa si falló\n\n## 🚀 Configuración\n\n### 1. Agregar el workflow\n\n```bash\n# Archivo ya creado en:\n.github/workflows/ci-failure-webhook.yml\n```\n\n### 2. Configurar GitHub Secrets (opcional, para webhook)\n\n```bash\n# En: Settings → Secrets → Actions\n\nSELF_HEALING_WEBHOOK_URL = https://your-agent-domain.com/webhooks/ci-failure\nSELF_HEALING_WEBHOOK_SECRET = your-secret-key-here\n```\n\n### 3. Asegurar permisos\n\n```yaml\npermissions:\n  issues: write        # Para crear issues\n  contents: read       # Para leer código/logs\n  actions: read        # Para leer workflow runs\n```\n\n### 4. Configurar triggers (opcional)\n\n```yaml\non:\n  workflow_run:\n    workflows: \n      - 'Tests'              # Detectar fallos de tests\n      - 'Integration Tests'  # Detectar fallos de integración\n      - 'Build'              # Detectar fallos de build\n    types: [completed]\n```\n\n## 🧪 Ejemplo Completo\n\n### Escenario: Test falla en main\n\n```\n1. Developer pushea a main\n   ↓\n2. GitHub Actions ejecuta workflow \"Tests\"\n   ↓\n3. test-unit job FALLA\n   ↓\n4. workflow_run evento trigger (conclusion = failure)\n   ↓\n5. ci-failure-webhook.yml se ejecuta\n   ├─ ✅ Branch \"main\" está permitida\n   ├─ ✅ No es rama de test\n   ├─ ✅ No existe issue duplicado\n   ↓\n6. Issue creado:\n   Título: \"🔴 Self-Healing: test-unit failed on branch main\"\n   Labels: [self-healing-candidate, auto-healing, ci-failure]\n   ↓\n7. Webhook enviado al agente IA\n   ↓\n8. Agente IA recibe notificación\n   ├─ Descarga issue #1234\n   ├─ Analiza logs\n   ├─ Diagnostica: \"Off-by-one error\"\n   ├─ Genera fix\n   ├─ Crea rama fix/ai-heal-1234\n   ├─ Crea PR #456\n   └─ Espera revisión humana\n```\n\n### Escenario: Duplicate trigger (5 minutos después)\n\n```\n1. Developer re-pushea el mismo commit\n   ↓\n2. GitHub Actions ejecuta workflow \"Tests\" de nuevo\n   ↓\n3. test-unit job FALLA de nuevo\n   ↓\n4. ci-failure-webhook.yml se ejecuta\n   ├─ ✅ Branch check: PASS\n   ├─ ✅ Pattern check: PASS\n   ├─ ❌ Duplicate check: FALLA\n   │   (Issue #1234 encontrado, creado hace 3 minutos)\n   ↓\n5. ⏭️ SKIP - No create issue\n```\n\n### Escenario: Test branch\n\n```\n1. Developer pushea a test/my-feature\n   ↓\n2. GitHub Actions ejecuta workflow \"Tests\"\n   ↓\n3. test-unit job FALLA\n   ↓\n4. ci-failure-webhook.yml se ejecuta\n   ├─ ✅ Branch check: test/my-feature\n   ├─ ❌ Pattern check: FALLA (matches \"test/*\")\n   ↓\n5. ⏭️ SKIP - No create issue\n   (No queremos auto-healing en ramas de test)\n```\n\n## 📊 Monitoreo\n\n### Métricas a observar\n\n```yaml\nmetrics:\n  issues_created_per_day: Count([label:self-healing-candidate, created_at > now-1d])\n  duplicate_attempts: Count([skipped due to duplicate detection])\n  branch_filters_applied: Count([skipped due to branch filter])\n  webhook_failures: Count([webhook notifications failed])\n  average_time_to_issue_creation: Duration(start: workflow_run, end: issue_created)\n```\n\n### Dashboard queries\n\n```\n# Issues creados hoy\nis:issue label:self-healing-candidate created:2026-06-16\n\n# Issues pendientes análisis\nis:issue label:self-healing-candidate label:\"status: pending-analysis\"\n\n# Issues ya resueltos\nis:issue label:self-healing-candidate label:\"status: resolved\"\n```\n\n## 🐛 Troubleshooting\n\n### Issue no se creó pero workflow completó\n\n**Posibles causas**:\n1. Branch filtrada (no en ALLOWED_BRANCHES)\n2. Duplicate encontrado (< 5 minutos)\n3. GITHUB_TOKEN sin permisos\n4. Workflow no en la lista de triggers\n\n**Verificar**:\n```bash\n# Ver logs del workflow\nGitHub Actions → ci-failure-webhook → Run details\n```\n\n### Webhook no se envía\n\n**Posibles causas**:\n1. SELF_HEALING_WEBHOOK_URL no configurado\n2. Secret mal configurado\n3. Firewall bloqueando\n\n**Verificar**:\n```bash\n# En workflow logs, buscar:\n\"Webhook URL not configured\" OR \"Webhook error\"\n```\n\n### Demasiados issues duplicados\n\n**Causa**: Ventana de 5 minutos muy corta para workflows que retrigger rápido\n\n**Solución**: Aumentar ventana en `deduplicate` step:\n```javascript\nconst fiveMinutesAgo = new Date(now.getTime() - 15 * 60000); // 15 minutos\n```\n\n## 📚 Referencias\n\n- [GitHub Actions Workflow Events](https://docs.github.com/en/actions/using-workflows/events-that-trigger-workflows#workflow_run)\n- [GitHub Script Action](https://github.com/actions/github-script)\n- [GitHub REST API - Issues](https://docs.github.com/en/rest/issues)\n- [GitHub REST API - Actions](https://docs.github.com/en/rest/actions)\n\n---\n\n**Última actualización**: Junio 2026\n"
