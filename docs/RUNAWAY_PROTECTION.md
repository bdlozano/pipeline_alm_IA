# Self-Healing CI - Protecciones contra Bucles Infinitos\n\n## 🔄 Problema: Comportamiento Descontrolado\n\nSin protecciones adecuadas, un agente de IA podría:\n\n```\n❌ Crear múltiples PRs para el mismo error\n❌ Hacer cambios que generan más errores\n❌ Entrar en bucle infinito de reintentos\n❌ Usar demasiados recursos (API, compute, dinero)\n❌ Crashear el sistema con cambios inválidos\n```\n\n---\n\n## 📊 Protecciones Implementadas\n\n### 1. DEDUPLICACIÓN - Evitar duplicados del mismo error\n\n| Protección | Mecanismo | Implementación |\n|-----------|-----------|----------------|\n| **Issue Dedup** | Un issue por problema único | Buscar issue existente con mismo commit + workflow |\n| **Ventana de 5 min** | Evita crear issues si ya existe reciente | `IF created_at > NOW - 5 minutes AND body.contains(commitSha)` |\n| **PR Dedup** | Un PR por análisis | Chequear si rama `fix/ai-heal-{issue_id}` ya existe |\n| **Commit Hash** | Firmar correcciones para rastrear | Usar commit message: `[ai-fix-#123]` para tracking |\n\n**Configuración JSON**:\n```json\n{\n  \"deduplication\": {\n    \"issue_search_window_minutes\": 5,\n    \"check_recent_commits\": true,\n    \"max_prs_per_issue\": 1,\n    \"use_idempotent_branch_naming\": true,\n    \"branch_pattern\": \"fix/ai-heal-{issue_id}\"\n  }\n}\n```\n\n**Pseudocódigo**:\n```python\ndef create_issue_if_not_duplicate(error_info):\n    # Buscar issues recientes con el mismo problema\n    similar_issues = search_issues(\n        q=f'label:self-healing-candidate \\\n            workflow:{error_info.workflow} \\\n            branch:{error_info.branch} \\\n            created:>{now - 5min}',\n        order='desc',\n        per_page=5\n    )\n    \n    # Verificar si alguno es para el mismo commit\n    for issue in similar_issues:\n        if error_info.commit_sha in issue.body:\n            logger.info(f\"Issue duplicado encontrado: #{issue.number}\")\n            return None  # No crear\n    \n    # Crear issue nuevo\n    return create_issue(error_info)\n```\n\n---\n\n### 2. LÍMITE DE REINTENTOS - Máximo de intentos por issue\n\n| Protección | Mecanismo | Límite |\n|-----------|-----------|--------|\n| **Retry Count** | Contar intentos por issue | Máximo 3 intentos por issue |\n| **Retry Backoff** | Esperar más tiempo entre intentos | Exponencial: 1min, 5min, 15min |\n| **Timeout Global** | Tiempo máximo para resolver | 4 horas máximo por issue |\n| **Test Failure Threshold** | Si tests fallan N veces | Si tests fallan 3x → escalada |\n\n**Configuración JSON**:\n```json\n{\n  \"retry_limits\": {\n    \"max_retries_per_issue\": 3,\n    \"backoff_strategy\": \"exponential\",\n    \"backoff_minutes\": [1, 5, 15],\n    \"global_timeout_hours\": 4,\n    \"max_test_failures\": 3,\n    \"retry_on_error_types\": [\n      \"test_failure\",\n      \"partial_fix\"\n    ],\n    \"no_retry_on_error_types\": [\n      \"permission_denied\",\n      \"rate_limited\",\n      \"invalid_fix\"\n    ]\n  }\n}\n```\n\n**Pseudocódigo**:\n```python\ndef should_retry_analysis(issue_id):\n    history = get_issue_history(issue_id)\n    \n    # Contar intentos\n    retry_count = sum(1 for h in history if h.action == 'analysis_attempt')\n    \n    if retry_count >= MAX_RETRIES:\n        logger.warning(f\"Issue #{issue_id}: max retries reached\")\n        escalate_to_human(issue_id, \"Max retries exceeded\")\n        return False\n    \n    # Calcular backoff\n    last_attempt = history[-1].timestamp\n    wait_time = calculate_backoff(retry_count)\n    \n    if (now - last_attempt) < wait_time:\n        logger.info(f\"Waiting {wait_time}m before retry...\")\n        return False\n    \n    # Verificar timeout global\n    total_time = now - history[0].timestamp\n    if total_time > GLOBAL_TIMEOUT:\n        logger.error(f\"Global timeout reached for issue #{issue_id}\")\n        escalate_to_human(issue_id, \"Global timeout\")\n        return False\n    \n    return True\n\ndef calculate_backoff(attempt_number):\n    # Exponential backoff: 1, 5, 15, 30...\n    return min(2 ** (attempt_number - 1), 60)  # Max 60 minutos\n```\n\n**Tabla de Backoff**:\n```\nIntento | Esperar | Timestamp Lógico\n--------|---------|------------------\n   1    | 1 min   | 14:30 → 14:31 (retry)\n   2    | 5 min   | 14:31 → 14:36 (retry)\n   3    | 15 min  | 14:36 → 14:51 (retry)\n   4    | → ESCALADA A HUMANO\n```\n\n---\n\n### 3. DETECCIÓN DE BUCLES - Identificar si está en loop\n\n| Señal | Indicador de Loop | Acción |\n|-------|---|---|\n| **Mismo error repetido** | Test falla con idéntico mensaje | Marcar como loop, investigar más |\n| **Error nuevo tras fix** | Fix aplicado, nuevo error diferente | Cambio causó regression, revisar fix |\n| **PR rechazado múltiples veces** | PR rechazado 2x con mismo feedback | Feedback no se implementó, escalar |\n| **Patrón cíclico** | Error A → Fix genera Error B → Fix B genera Error A | Detecto loop, parar |\n\n**Configuración JSON**:\n```json\n{\n  \"loop_detection\": {\n    \"detect_identical_errors\": true,\n    \"identical_error_threshold\": 2,\n    \"detect_error_patterns\": true,\n    \"error_signature_hash\": true,\n    \"max_unique_errors_per_issue\": 5,\n    \"track_error_history_depth\": 10,\n    \"loop_detection_enabled\": true,\n    \"escalate_on_loop\": true\n  }\n}\n```\n\n**Pseudocódigo**:\n```python\ndef detect_loop(issue_id):\n    history = get_issue_history(issue_id)\n    error_signatures = []\n    \n    for attempt in history:\n        if attempt.action == 'test_failed':\n            # Crear firma del error (hash del mensaje)\n            sig = hash_error_signature(attempt.error_message)\n            error_signatures.append(sig)\n    \n    # Detectar pattern cíclico (A → B → A → B)\n    if len(error_signatures) >= 4:\n        # Últimas 4 firmas\n        recent = error_signatures[-4:]\n        if recent[0] == recent[2] and recent[1] == recent[3]:\n            logger.error(\"Loop detectado: A→B→A→B\")\n            return \"cycle_loop\"\n    \n    # Detectar mismo error repetido\n    if len(error_signatures) >= 2:\n        if error_signatures[-1] == error_signatures[-2]:\n            count = sum(1 for sig in error_signatures if sig == error_signatures[-1])\n            if count >= 2:\n                logger.error(f\"Mismo error {count} veces\")\n                return \"same_error_loop\"\n    \n    # Detectar explosion de errores (nuevo error cada vez)\n    unique_errors = len(set(error_signatures))\n    if unique_errors > MAX_UNIQUE_ERRORS:\n        logger.error(f\"Demasiados errores diferentes: {unique_errors}\")\n        return \"error_explosion\"\n    \n    return None  # No es loop\n```\n\n**Detección de Bucle en Acción**:\n```\nIntento 1: npm ERR! 404 - uuid-v7 not found\n           ↓\n           Fix: actualizar package.json\n           ↓\nIntento 2: npm ERR! 404 - @types/uuid not found\n           ↓\n           Fix: instalar @types/uuid\n           ↓\nIntento 3: npm ERR! 404 - uuid-v7 not found  ← ¡BUCLE DETECTADO!\n           ↓\n           🚨 ESCALADA A HUMANO: \"Loop detectado. Problema raíz no entendido.\"\n```\n\n---\n\n### 4. BACKOFF EXPONENCIAL - Esperar más tiempo entre intentos\n\n**Estrategias**:\n\n```\nEstilos de Backoff:\n\n1. LINEAL (simple, no recomendado):\n   Espera: 1, 2, 3, 4, 5... minutos\n   ❌ Satura rápido\n\n2. EXPONENCIAL (recomendado):\n   Espera: 1, 2, 4, 8, 16... minutos\n   ✅ Escalada controlada\n\n3. EXPONENCIAL + JITTER (mejor):\n   Espera: 1±0.1, 2±0.2, 4±0.4... minutos\n   ✅ Evita thundering herd\n\n4. FIBONACCI (balanced):\n   Espera: 1, 1, 2, 3, 5, 8... minutos\n   ✅ Intermedio entre lineal y exponencial\n```\n\n**Implementación Exponencial + Jitter**:\n```python\nimport random\n\ndef calculate_backoff_with_jitter(attempt_number):\n    # Exponential backoff\n    base_wait = 2 ** (attempt_number - 1)  # 1, 2, 4, 8...\n    \n    # Cap at max\n    base_wait = min(base_wait, 60)  # Max 60 minutos\n    \n    # Add jitter (±10%)\n    jitter = base_wait * random.uniform(-0.1, 0.1)\n    final_wait = base_wait + jitter\n    \n    return max(1, final_wait)  # Min 1 minuto\n\n# Ejemplo:\nfor attempt in range(1, 6):\n    wait = calculate_backoff_with_jitter(attempt)\n    print(f\"Intento {attempt}: esperar {wait:.1f} minutos\")\n\n# Output:\n# Intento 1: esperar 0.9 minutos\n# Intento 2: esperar 2.1 minutos\n# Intento 3: esperar 3.8 minutos\n# Intento 4: esperar 8.2 minutos\n# Intento 5: esperar 55.3 minutos\n```\n\n**Configuración JSON**:\n```json\n{\n  \"backoff\": {\n    \"strategy\": \"exponential_with_jitter\",\n    \"base_multiplier\": 2,\n    \"jitter_percentage\": 10,\n    \"max_wait_minutes\": 60,\n    \"min_wait_minutes\": 1\n  }\n}\n```\n\n---\n\n### 5.
